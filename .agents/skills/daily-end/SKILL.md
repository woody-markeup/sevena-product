# Daily End Skill (每日下班前同步)

此技能用於每日下班前，協助使用者同步今日的工作進度，並產出結構化、分為三層的 Report 報告。

## 執行流程與指令

當此技能被觸發時，請依序執行以下步驟：

### 1. 取得基礎資訊
- **日期**：獲取當前日期，格式為 `YYYYMMDD` (例如 `20260702`)。
- **使用者名稱**：預設為 `Woody`（可從 git config 中的 user.name 取得，或直接使用 `Woody`）。

### 2. 載入今日的 To-do 記錄
- 讀取 `daily_sync/[YYYYMMDD]_todo.md` 或 `[YYYYMMDD]_todo.json`。
- 如果找不到，請引導使用者提供今日任務清單。

### 3. 分析今日 Git 異動並對齊進度
- **[重要] 檢查所有 Git 專案**：除了主專案，必須主動走訪專案中所有包含 `.git` 的獨立子專案目錄（如 `sevena/`, `sevena-backend/`, `sevena-teacher/` 等）。
- 於各專案目錄執行：
  - 查詢今日的所有 commit：`git log --since="midnight" --oneline`，並使用 `git show --stat <commit>` 或 `git log --stat` 分析其具體修改的檔案。
  - 查詢工作區尚未 commit 的變動：`git status --porcelain` 或 `git status`。
- 逐一對齊並確認每個項目的完成狀態（如「已完成」、「進行中」、「已取消」等），並記錄其**具體開發的功能、修改的 Swift/代碼檔案與規格書路徑**。

### 4. 生成並輸出三層結構 of Daily Report
將結果整理為以下三層結構。**注意：此輸出最後不要附上任何使用了哪些 Agent 技能或規定的備註。**

*   **第一層：回報事項** (給外人看的標準格式，列出所有任務與其完成狀態。**必須寫出具體完成的代碼功能模組、修改的 Swift/代碼檔案路徑與成果細節，不得只列項目標題與狀態**)
*   **第二層：來源事項** (說明這些任務是在哪裡檢查到的，提供關聯的 Commit ID 與檔案路徑)
*   **第三層：對齊時新加上** (記錄在對齊時新加上的任務之完成狀態與具體成果)

```markdown
----下班前------
[使用者名稱] [YYYYMMDD] Report
- 項目1 [狀態]
  * 實質開發工作與修改檔案 (例如: 實作遠端控制通道, 修改 [FeatureWheel.swift](file://...))
- 項目2 [狀態]
  * 實質開發工作與修改檔案
- 新項目A [狀態]

來源事項：
- 項目1：來源說明 (檔案路徑/Commit ID/修改檔案清單)
- 項目2：來源說明

對齊時新加上：
- 新項目A：新加上的理由，狀態為 [狀態]，具體成果為 [成果]
```

### 5. 保存記錄至回報資料夾
- 將上述三層結構的完整 Markdown 內容，保存至主專案根目錄的回報資料夾：
  `daily_sync/[YYYYMMDD]_report.md`。
