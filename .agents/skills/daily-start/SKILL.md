# Daily Start Skill (每日開始工作同步)

此技能用於每日上班後，協助使用者同步與整理今日的 To-do 項目，自動參考相關變更，並產出結構化、分為三層的對齊報告。

## 執行流程與指令

當此技能被觸發時，請依序執行以下步驟：

### 1. 取得基礎資訊
- **日期**：獲取當前日期，格式為 `YYYYMMDD` (例如 `20260702`)。
- **使用者名稱**：預設為 `Woody`（可從 git config 中的 user.name 取得，或直接使用 `Woody`）。

### 2. 檢索 Spec 變更與 Git 提交
檢查以下來源以提供今日的任務推薦：
- `sevena/openspec/changes/` 下進行中的變更與 tasks 檔案。
- 根目錄 `contracts/` 下的契約檔案與 `docs/PROGRESS.md`。
- 本日已 commit 的項目。
- **[重要] 檢查子專案**：主動走訪並檢索主專案與所有獨立 Git 子專案目錄（如 `sevena/`, `sevena-backend/`, `sevena-teacher/` 等）的 Git 變更、未追蹤檔案與本日 Commit，避免遺漏子專案中產生的開發任務。

### 3. 與使用者同步今日任務
引導使用者確認今日計畫，確認哪些是原本計畫，哪些是與 Spec 對齊後產生的，哪些是與 Agent 對齊時「新加上」的。

### 4. 生成並輸出三層結構的 Daily To-do
將今日任務整理成以下三層結構。**注意：此輸出最後不要附上任何使用了哪些 Agent 技能或規定的備註。**

*   **第一層：回報事項** (給外人看的標準格式，僅列出項目與工時，**絕對不要標註是否完成**)
*   **第二層：來源事項** (說明 these 任務是在哪裡檢查到的，需提供檔案路徑、Commit ID 或契約來源)
*   **第三層：對齊時新加上** (原本不在計劃中，但在與 Agent 對齊討論時新加入的任務)

```markdown
-----上班後------
[使用者名稱] [YYYYMMDD] To-do
-項目1 [估時]
-項目2 [估時]
-新項目A [估時]

來源事項：
- 項目1：來源說明 (檔案路徑/Commit)
- 項目2：來源說明

對齊時新加上：
- 新項目A：新加上的理由與討論背景
```

### 5. 保存記錄至回報資料夾
將上述三層結構的完整 Markdown 內容，保存至主專案根目錄的回報資料夾：
`daily_sync/[YYYYMMDD]_todo.md`。
同時，可將 todos 清單以 JSON 格式儲存到 `daily_sync/[YYYYMMDD]_todo.json` 中以供後續程式化讀取。
